## 观点

- 设计是“约束下的选择”：先定义体验目标与约束（时间、复杂度、学习成本），再谈系统与机制
- 反馈回路比功能堆叠更重要：让玩家快速理解“我做了什么 → 世界发生了什么”，体验才会成立
- 平衡的核心是节奏：强弱不是问题，节奏断裂才是问题（挑战—休息—奖励的循环）
- 工程上要为迭代服务：可调参数、可视化调试、快速回滚，比一次性做对更关键

## 参考素材

- 《The Art of Game Design》：把体验拆解成可讨论的“镜头”，适合构建设计语言
- 《A Theory of Fun for Game Design》：解释“为什么会觉得有趣”，帮助避免凭感觉做决定
- 《Game Programming Patterns》：把复杂系统做成可扩展的结构，适合反复翻
